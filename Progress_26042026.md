## 1. บทนำและเป้าหมายโครงการ

โครงการนี้นำระบบ **SEA (Segment, Embed, and Align)** มาประยุกต์ใช้กับ **ภาษามือไทย (Thai Sign Language — TSL)** โดยเป้าหมายหลักคือ:

- **จัดตำแหน่งคำบรรยาย (subtitle) ให้ตรงกับช่วงเวลาที่ผู้แปลภาษามือแสดงท่าทาง** ในวิดีโอจริง
- ทดสอบว่า SEA ซึ่งพัฒนาสำหรับ BSL (British Sign Language) สามารถ **ทำงานข้ามภาษา (cross-lingual)** มาใช้กับ TSL ได้หรือไม่
- เปรียบเทียบประสิทธิภาพของ **โมเดล embedding หลายตัว** (BSL, Multilingual, ASL) กับข้อมูลภาษามือไทย
- สร้าง **เครื่องมือ pipeline ครบวงจร** ที่สามารถนำไปใช้ซ้ำกับวิดีโอภาษามือไทยตัวอื่นได้

---

### 🎯 เป้าหมายหลัก 2 งาน (Main Tasks)

#### งานที่ 1: CC (timestamp) → CC_Aligned (timestamp)

> เลื่อนเวลาของคำบรรยาย (CC) ให้ไปตรงกับจังหวะที่ผู้แปลภาษามือแสดงท่าทาง

**สถานะ: ✅ เสร็จสมบูรณ์ + Post-processing**

- **C_MULTI** (Multilingual + Gloss text) คือ experiment ที่ดีที่สุด
- Mean offset **+0.49s**, **80% within ±1s**, **99% within ±3s** *(วัดบน 69 cues ที่ match ได้ — ดู §14 หมายเหตุ)*
- หลัง overlap fix: **0% overlap** (จาก ~88%)
- ครบ 7 experiments แล้วทั้งก่อนและหลัง post-processing

#### งานที่ 2: Gloss → Gloss Labeling

> จัดตำแหน่ง gloss ระดับประโยค (119 entries) ลงไปเป็น annotation รายท่ามือแต่ละท่า (852 entries)

**สถานะ: ✅ Prototype ทำงานสำเร็จ (เปลี่ยนจาก ⚠️ planned)**

- **Mean IoU = 0.49** เทียบกับ ground truth (852 entries)
- **48.4% ของ predictions มี IoU ≥ 0.5**
- **77.0% มี IoU ≥ 0.3**
- **65.1% text-match** บนคู่ที่ overlap กัน
- ครอบคลุม 852/852 labels โดยไม่ต้อง uniform fallback แม้แต่ประโยคเดียว
- รายละเอียด pipeline + DP recurrence + ผลทดลอง อยู่ใน [§12](#12-ขั้นตอนที่-7--task-2-gloss-labeling-prototype)

## 12. ขั้นตอนที่ 7 — Task 2: Gloss Labeling Prototype

**สถานะ: ✅ Prototype ทำงานได้ — Mean IoU = 0.49**

### 12.1 ภาพรวมแนวคิด

```text
Gloss sentence (start_s, end_s, "ผายมือ เด็ก เรียน")
        │
        ├─► tokenize on whitespace      → ["ผายมือ", "เด็ก", "เรียน"]
        │
        ├─► restrict candidate SIGN segments
        │     mid in [start_s, end_s]   (±0.5s pad if empty)
        │
        ├─► embed each token via SignCLIP multilingual text encoder
        │     → token_embs (T × 768), cached to .npz
        │
        ├─► sim_matrix (T × K) = cosine + row softmax
        │
        ├─► monotonic DP per sentence
        │     dp[t][j] = min over k of {
        │         dp[t-1][k-1]
        │       + (-Σ sim[t-1, k-1..j-1])
        │       + gap_penalty × inter_segment_gap_total
        │       + coverage_penalty × |group_dur − sentence_dur/T|
        │     }
        │
        └─► backtrack → assign each token to (k_start, k_end) range
              → emit (seg[k_start].start, seg[k_end].end, token)
```

### 12.2 DP recurrence (per sentence)

```
State: dp[t][j] = min cost assigning tokens 1..t with token t ending at segment j.

Transition (1 ≤ t ≤ T, t ≤ j ≤ K):
  dp[t][j] = min over k ∈ [t .. j] of {
      dp[t-1][k-1]
      + (-cum_sim[t-1, j] + cum_sim[t-1, k-1])           ← negative similarity
      + gap_penalty × Σ (seg_starts[k..j-1] − seg_ends[k-1..j-2])
      + coverage_penalty × | (seg_ends[j-1] − seg_starts[k-1]) − sentence_dur/T |
  }

Boundary: dp[0][0] = 0, all others = +∞
Final:    j* = argmin over j ∈ [T..K] of dp[T][j]
Backtrack: prev[t][j] gives k* ⇒ ranges = [(k*-1, j-1)] from t=T down to 1
```

**Complexity:** O(T × K²) per sentence. With T~7, K~30, this finishes 119 sentences in <1 second.

| Flag | ค่าเริ่มต้น | ผลกระทบ |

| `--language-tag` | `<en> <bfi>` | language tag prepended (เหมือน C_MULTI) |
| `--gap-penalty` | 2.0 | ↑ ลด group ที่มี gap |
| `--coverage-penalty` | 0.5 | ↑ บังคับให้ duration เฉลี่ยตาม sentence_dur/T |

## 16. ปัญหาที่พบและวิธีแก้ไข

### 16.1 mediapipe version ไม่ถูกต้อง
**ปัญหา:** mediapipe 0.10.22+ เปลี่ยน API
**แก้ไข:** ล็อก `mediapipe==0.10.21`

### 16.2 UnicodeDecodeError: video_ids.txt
**ปัญหา:** `echo 04 > video_ids.txt` บน Windows สร้าง UTF-16 BOM
**แก้ไข:** ใช้ Python เขียน UTF-8

### 16.3 FileNotFoundError: shlex.quote() บน Windows
**ปัญหา:** `shlex.quote()` ใส่ single quote ที่ cmd.exe ไม่รู้จัก
**แก้ไข:** เปลี่ยน `segmentation.py` เป็น `subprocess.run(shlex.split(cmd), shell=False)`

### 16.4 unrecognized argument: --subtitle_dir
**แก้ไข:** ใช้ `--pr_sub_path` แทน

### 16.5 ModuleNotFoundError: numba, beartype
**แก้ไข:** `pip install numba beartype tqdm scikit-learn tabulate matplotlib`

### 16.6 PyTorch ไม่ detect GPU (RTX 5060 Ti Blackwell)
**แก้ไข:** ติดตั้ง PyTorch cu128

### 16.7 Overlap สูง ~88% ในผล alignment
**แก้ไข:** ใช้ `fix_overlap_vtt.py` (clamp end time) → 0%

### 🆕 16.8 SignCLIP yaml อ้างอิง checkpoint ด้วย relative path
**ปัญหา:** `align_gloss_labels.py` พยายามโหลด `runs/retri_v1_1/baseline_temporal/checkpoint_best.pt` แต่หาไม่เจอเพราะ path เป็น relative
**สาเหตุ:** `MMPTModel.from_pretrained()` ใช้ relative path เทียบกับ cwd
**แก้ไข:** `os.chdir(fairseq_signclip/examples/MMPT)` ก่อนเรียก `load_model()` แล้ว `chdir` กลับหลังเสร็จ — โค้ด pattern อยู่ใน `align_gloss_labels.embed_tokens_cached()`

### 🆕 16.10 evaluate_all.py — gt_by_text เก็บแค่ first occurrence
**ปัญหา (known limitation):** `gt_by_text` dict ใน `evaluate_all.py` ใช้ `if key not in gt_by_text` guard ทำให้ถ้า CC_Aligned มีข้อความซ้ำกัน 2 ครั้ง จะเก็บแค่ entry แรก — entry ที่ 2 จะหา match ไม่ได้
**ผลกระทบ:** ตรวจสอบแล้วมี duplicate 1 รายการ (`"เราไปดูหลักและวิธีการเปรียบเทียบกันนะคะ"`) → กระทบ 1 cue เท่านั้น ในทางปฏิบัติเล็กน้อย
**แก้ไขในอนาคต:** เปลี่ยน `gt_by_text` ให้เก็บ `list` ต่อ key แล้ว iterate หาคู่ที่ดีที่สุด

### 🆕 16.9 Gloss tokens ภาษาไทยกับ SignCLIP — กังวลเริ่มต้นไม่เป็นจริง
**ความกังวล (ก่อนทดลอง):** SignCLIP multilingual encoder อาจไม่เข้าใจ Thai tokens ระดับคำ → IoU ต่ำ
**ผลที่ได้จริง:** **Mean IoU = 0.49** (เกิน sanity bar 0.30 มาก) เพราะ:
- Tokens ส่วนใหญ่เป็น parenthesized notation เช่น `(ผายมือ)` ที่ใช้บ่อยมาก
- DP ใช้ temporal constraint ของ Gloss sentence → search space เล็กลงมาก
- Word-level embedding ใช้ language tag เดียวกับ C_MULTI ที่ผ่านมา
- **ไม่ต้องทำ gloss-to-English mapping** ในเฟสแรกนี้
